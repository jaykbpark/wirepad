# Agent Instructions

## Tooling Policy

- Use standard shell and git tooling for this repository.
- Prefer local CLI workflows and file-based edits.
- Avoid IDE-specific or simulator-specific MCP integrations for this project.

## Git Policy

- Push directly to `main`.
- Keep commits atomic with bracketed prefixes:
  - `[feat]`
  - `[bug]`
  - `[docs]`
  - `[chore]`
  - `[refactor]`
  - `[test]`
